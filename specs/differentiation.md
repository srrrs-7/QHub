# Differentiation Strategy

## GitHub との対比: 自然言語ネイティブな管理

| 側面 | GitHub (コード) | PromptLab (自然言語) |
|------|----------------|---------------------|
| **差分** | 行単位テキスト diff | Semantic Diff (意味的理解) |
| **テスト** | 自動テスト pass/fail | 品質評価スコア (0-100) |
| **デプロイ** | CI/CD パイプライン | Draft → Review → Production |
| **レビュー** | コードロジックの正しさ | 意図・トーン・効果の評価 |
| **品質指標** | テストカバレッジ | 品質スコア + 業界ベンチマーク |
| **アドバイザー** | Copilot (コード補完) | **Consulting Chat (改善提案)** |

---

## 4 つの差別化機能

### 1. Semantic Diff (意味的差分)

**競合の現状**: 全競合がテキスト diff のみ。

**PromptLab**: テキスト diff + LLM による意味的変更サマリー + 影響予測。

```
📝 Text Diff:
- You are a helpful assistant.
+ You are a helpful coding assistant specializing in {{language}}.

🧠 Semantic Summary:
・汎用アシスタントからコーディング特化に変更
・新しい変数 {{language}} を追加
・回答のスコープを限定する方向の変更

📊 Impact Prediction:
・専門性: ↑ | 汎用性: ↓ | トークン: +15
```

### 2. Prompt Linting (自然言語の静的解析)

**競合の現状**: Promptfoo がセキュリティスキャン(レッドチーム)を提供。構造的品質 lint はゼロ。

**PromptLab**: ルールベース + LLM ベースの品質チェック。

| ルール | 種別 | 説明 |
|--------|------|------|
| variable-unused | ルール | 定義された変数がテンプレートで未使用 |
| variable-undefined | ルール | テンプレートの変数が未定義 |
| define-output-format | ルール | 出力形式の指定がない |
| excessive-length | ルール | 推奨トークン数を超過 |
| no-vague-instruction | LLM | 曖昧な指示を検出 |
| missing-constraints | LLM | 制約の不足を検出 |
| prompt-injection-risk | LLM | インジェクションリスクを検出 |

### 3. Consulting Chat (業界特化 AI コンサルタント)

**競合の現状**: どの競合も提供していない。完全なホワイトスペース。

**PromptLab**: 蓄積データ × 業界ナレッジ × LLM = パーソナライズされた改善コンサルタント。

**なぜ競合にできないか**:
- 観測ツール (Langfuse, Arize) はデータはあるがプロンプト管理が弱い → 改善提案の文脈が不足
- バージョン管理ツール (PromptHub) はプロンプトはあるが品質データが薄い → データドリブンな提案ができない
- PromptLab は**バージョン + 品質データ + 業界知識**の三位一体で初めて実現する

**コンサルの 4 つのモード**:

| モード | 入力 | 出力 |
|--------|------|------|
| 改善提案 | プロンプト + 品質データ | 具体的な改善案 + 新バージョン作成 |
| ベンチマーク比較 | 組織スコア | 業界比較 + 改善優先順位 |
| コンプライアンス | プロンプト + 業界 | 準拠チェック結果 + 修正提案 |
| 一般相談 | 自由質問 | プロンプトエンジニアリングのアドバイス |

### 4. アウトカム連動バージョン管理

**競合の現状**:
- PromptHub: バージョン管理は強いが品質データと未連動
- Braintrust: 品質ゲートはあるがバージョン管理 UI が弱い
- Langfuse: 観測は強いがバージョン管理がサブ機能

**PromptLab**: バージョンと品質データを**同一画面で統合**。

```
v5 (Production) ─── Score: 89.3 ─── 450 executions ─── $2.55/1K
v4 (Archived)   ─── Score: 78.2 ─── 800 executions ─── $2.85/1K
v3 (Archived)   ─── Score: 72.1 ─── 320 executions ─── $3.20/1K

📈 Quality Trend: 72.1 → 78.2 → 89.3
💰 Cost Trend:   $3.20 → $2.85 → $2.55/1K
```

## フライホイール: データ → 知識 → 価値

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ユーザーがプロンプトを管理・改善 (Layer 1)             │
│        │                                                │
│        ▼                                                │
│   実行ログ・評価データが蓄積                             │
│        │                                                │
│        ▼                                                │
│   コンサルチャットがより精度の高い提案 (Layer 2)         │
│        │                                                │
│        ▼                                                │
│   プロンプト品質が向上 → ユーザーの成功体験              │
│        │                                                │
│        ▼                                                │
│   口コミ → 新規ユーザー → さらにデータ蓄積              │
│        │                                                │
│        └──────────── ♻️ ──────────────────────────┘     │
│                                                         │
│   + 匿名ベンチマーク (オプトイン) → 業界全体のナレッジ化 │
│     → コンサルの価値がさらに向上                         │
└─────────────────────────────────────────────────────────┘
```

**このフライホイールこそが最大の参入障壁**。
データが溜まるほどコンサルが賢くなり、後発が追いつけなくなる。
